ROOT CAUSE IDENTIFIED
════════════════════════════════════════════════════════════════════════════

The capture_*.txt files are NOT actual CAN bus captures.
They are GENERATION PLANS - test scripts defining what to SEND, not results.

File Format:
  Type=D = TX frames we GENERATE (what we SEND)
  Type=R = Remote/Echo frames (part of test plan)  
  Type=RX = MISSING! (Where ECU responses should go)

Why 0x130 appears constantly:
  It's just another TX ID in the fuzzing sequence, not a response!
  The file generates 4 variants per ID, eventually cycling through all IDs.
  No responses were captured because the code NEVER SAVED RX data to file.
